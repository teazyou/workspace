# Repository Index ‚Äî `~/workspace`

> Centralized macOS environment configuration: app configs, shell setup, helper functions, and install/system scripts. This repo is the **source of truth**; most files are symlinked into the locations each app expects (`repo file ‚Üê system path`). See `setup_symlinks.sh` for the symlink wiring (it covers 6 of the 10 documented links ‚Äî the rest, incl. the AutoRaise pair and the nordvpn LaunchAgent plist, are wired out-of-band; see `docs/install/bootstrap-flow.md`).
>
> **KEEP THIS CURRENT:** when you add, move, remove, or rename a file, or change a symlink target, update the matching entry here in the same change.

All prose/context documentation lives in `docs/`, subfoldered by area (`docs/window-manager/`, `docs/install/`, `docs/scripts/`). Read the relevant guide there before acting on a topic. Guides + reference notes belong in `docs/`; this index + CLAUDE.md point into it.

```
configs/    app configs (aerospace, sketchybar, borders, autoraise, vscode, iterm2)
docs/       prose context guides, subfoldered by area (window-manager, install, scripts)
zsh/        ~/.zshrc + sourced configs and aliases
scripts/    install/system scripts (installs/, git/, checkpoint, obsi)
functions/  shared SH helpers sourced by scripts
logs/       runtime logs + checkpoint state (gitignored)
```

## docs

- `docs/install/bootstrap-flow.md` ‚Äî fresh-Mac install & bootstrap orchestration: the two-stage `bootstrap.sh`‚Üí`installation.sh` handoff, the 17-step ordering + dependency constraints, per-step idempotency checks, and the gaps the flow does NOT wire (AutoRaise binary/config/daemon). *Read for any fresh-Mac setup or `scripts/installs/` change.*
- `docs/scripts/checkpoint-and-git-scripts.md` ‚Äî the checkpoint auto-commit/auto-push system (the inverted hourly gate vs the unconditional manual runner) and the transparent `git push` wrapper (the zsh `git()` function that reroutes every push through `gpush.sh`). *Read when touching `scripts/git/` or the `checkpoint_*` scripts.*

## configs
App configuration; the window-manager stack (aerospace + sketchybar + borders + autoraise) is the most-edited area. Its authoritative guide is `docs/window-manager/guide-window-manager.md`.
- `configs/aerospace/aerospace.toml` ‚Äî AeroSpace tiling WM config (workspaces, keybindings, gaps) ‚Äî `‚Üê ~/.aerospace.toml`. Its startup `after-startup-command` sources `lib-paths.sh`, waits for the bar, runs `apply-display-profile.sh --force`, then resets the performance-mode `/tmp` state file and runs the toggle from clean state ‚Äî startup default: performance mode ON (minimal bar: spaces + calendar only); the bar itself draws on the main monitor only (sketchybarrc `display=main`). Service mode (alt+shift+;): `p` = performance mode. *Edit when: changing window-manager keybindings/layout. Run `aerospace reload-config` after.*
- `configs/aerospace/*.plist` ‚Äî LaunchAgent: `com.aerospace.display-profile.plist` (`‚Üê ~/Library/LaunchAgents/`, auto gap profile per display, StartInterval 60).
- `configs/aerospace/*.sh` ‚Äî WM helper scripts run by aerospace/LaunchAgents: `lib-paths.sh` (shared lib, below), `apply-display-profile.sh` (auto top-gaps **and** the workspace 7-9 monitor assignment ‚Äî flips between `'built-in.*'` and `'secondary'` per which display is main, so home/travel both work), `open-dock-app.sh`, `performance-mode.sh` (minimal-bar performance toggle, service-mode `p`, **default ON at every (re)start**: hides the resources/connectivity divisions + spacer0-1 and zeroes their members' update freqs; brackets stay drawing=on with BOTH paint layers ‚Äî fill + drop shadow ‚Äî toggled together, the empty-pill lesson; OFF restores drawing + the original freqs then forces `sketchybar --update`; does NOT touch the display-profile LaunchAgent).
- `configs/aerospace/lib-paths.sh` ‚Äî shared library **`source`d by the aerospace scripts above AND by aerospace.toml's startup line**: the single source of truth for the `/tmp` state-file path (`PERFORMANCE_MODE_STATE`) and `PLACEMENT_CAP_SECONDS=18` (open-dock-app's placement-enforcer cap). *Edit when: changing any cross-script path or timing constant.* (The empty-workspace watcher/MRU/grace/`aero()` stack was removed 2026-07 ‚Äî AeroSpace natively reveals the next focused window's workspace when the last window closes.)
- `configs/sketchybar/sketchybarrc` ‚Äî status-bar entry point (sources colors/icons/theme, loads items + plugins; bar `display=main` ‚Äî main monitor only, secondaries never draw it) ‚Äî dir `‚Üê ~/.config/sketchybar`. *Edit when: changing the status bar.*
- `configs/sketchybar/colors.sh`, `icons.sh` ‚Äî shared palette and icon glyphs. colors.sh's `BORDER_ACTIVE`/`PINK` (`0xffaa2222`) must stay in sync with `configs/borders/bordersrc` (its header says so).
- `configs/sketchybar/theme.sh` ‚Äî **visual template / single source of truth** for "division" geometry (corner radius, border, blur, drop shadow, `GROUP_GAP` between divisions, plus `DIVISION_PAD`/`ELEMENT_GAP` for inner edge + element spacing). Sourced by sketchybarrc before items; every bracket on both sides pulls from these tokens. *Edit when: restyling the bar's overall pill/division look (radius, shadow, spacing).*
- `configs/sketchybar/helpers/wifi_rssi.swift` ‚Äî tiny CoreWLAN Swift helper printing the current Wi-Fi link RSSI (no scan, no Location permission). plugins/wifi.sh compiles it on demand to the gitignored `helpers/wifi_rssi` binary for the strength icon (macOS 26 removed `airport`). *Edit when: changing what the helper reports.*
- `configs/sketchybar/helpers/wifi_rssi` ‚Äî the **compiled** Swift binary (gitignored; should exist locally ‚Äî plugins/wifi.sh builds it on first run).
- `configs/sketchybar/items/*.sh` ‚Äî one bar item definition per file, BUT not all are wired: only the **8 SOURCED items** are live (spaces, calendar, battery, ram, cpu, vpn, ethernet, wifi); `apple.sh`/`settings.sh` are commented and `front_app.sh`/`brew.sh`/`github.sh`/`spotify.sh` are not sourced (6 inert template leftovers). The audio division (volume + headset) was removed entirely 2026-07 ‚Äî its item/plugin files are deleted (icons.sh glyphs kept). Paddings come from theme.sh tokens. Several live items are state-driven: ethernet shows only when connected; vpn = native-IKEv2 NordVPN state (red+country label=connected, yellow=connecting, orange=`nord refresh` needed, grey=off; click = `plugins/vpn_click.sh` ‚Äî instant busy feedback (yellow "‚Ä¶") + click lock, then `nord toggle`; custom `vpn_change` event triggered by the `scripts/vpn/` stack); wifi = RSSI bars.
- `configs/sketchybar/plugins/*.sh` ‚Äî update/event scripts backing the items; **the live plugin set is ~12** and ~11 are dead legacy template leftovers. Live + load-bearing: `icon_map.sh` (maps apps to sketchybar-app-font glyphs, used by `aerospace.sh` for the spaces app icons), `aerospace.sh`, `date.sh`, `time.sh`, `wifi.sh` (RSSI strength icon), plus the per-item updaters (ethernet, ram, cpu, vpn, battery) and `vpn_click.sh` (vpn click handler: busy feedback + click lock ‚Üí `nord toggle`). *Do not assume a plugin is dead just because it's old ‚Äî icon_map/aerospace/date/time are live.*
- `configs/borders/bordersrc` ‚Äî JankyBorders window-border config ‚Äî dir `‚Üê ~/.config/borders`. `active_color=0xffaa2222` MUST stay in sync with colors.sh `BORDER_ACTIVE` (colors.sh's header literally says "keep in sync with configs/borders/bordersrc"). *Edit when: changing border colour/width/style ‚Äî mirror the colour to colors.sh.*
- `configs/autoraise/config` ‚Äî AutoRaise focus-follows-mouse tuning ‚Äî `‚Üê ~/.config/AutoRaise/config` (also lowercase `autoraise/`). The live config sets `requireMouseStop=false` (no rest/dwell model ‚Äî the coarse `pollMillis` IS the fly-over debounce) and `ignoreSpaceChanged=true`. *Edit when: tuning poll timing / fly-over debounce.*
- `configs/autoraise/com.autoraise.daemon.plist` ‚Äî AutoRaise LaunchAgent ‚Äî `‚Üê ~/Library/LaunchAgents/` (note: AutoRaise's binary, config symlink, and daemon are **not** installed/wired by the fresh-Mac flow; see `docs/install/bootstrap-flow.md`).
- `configs/vscode/settings.json` ‚Äî VS Code user settings ‚Äî `‚Üê ~/Library/Application Support/Code/User/settings.json`.
- `configs/iterm2/com.googlecode.iterm2.plist` ‚Äî iTerm2 prefs. Loaded via iTerm2's **"Load preferences from custom folder"** (a two-way LIVE file at `$APP_CONFIGS/iterm2` via `PrefsCustomFolder` + `LoadPrefsFromCustomFolder`, **NOT a symlink**), wired by `scripts/installs/install_iterm2.sh`. **Gotcha: iTerm2 must NOT be running when those defaults are written or it overwrites the plist on quit.** (The `recolor` random-tab-color helper lives in `zsh/configs/iterm2.zsh`.)
- `configs/claude/com.teazyou.claude-quota-keepalive.plist` ‚Äî **experimental** LaunchAgent (`‚Üê ~/Library/LaunchAgents/`, StartInterval 3600) firing `scripts/claude/quota_keepalive.sh` hourly. **Deliberately NOT wired into the install flow** (installation.sh / setup_symlinks.sh) ‚Äî loaded by hand via `launchctl bootstrap`; disable/remove instructions in the script header. *Edit when: changing the ping cadence.*
- `configs/nordvpn/com.teazyou.nordvpn-native.plist` ‚Äî native-NordVPN reconnect LaunchAgent (`‚Üê ~/Library/LaunchAgents/`, symlinked out-of-band, NOT in setup_symlinks.sh): `RunAtLoad` + `WatchPaths` on resolv.conf fire the one-shot `scripts/vpn/nord-connect.sh` on login/wake/network change ‚Äî **no StartInterval, no KeepAlive, zero resident processes**. See `docs/vpn/guide-nordvpn-native.md`. *Edit when: changing what events trigger reconnection.*
- `configs/dot-obsidian/` ‚Äî central Obsidian config (plugins, snippets, hotkeys, appearance, *.json settings). Not symlinked as a whole; the `obsi` launcher (`scripts/obsi`) plants per-file symlinks from each vault's `.obsidian/` to entries here (the per-file-symlink/BLACKLIST model is documented in `scripts/obsi`). Per-vault runtime state (workspace.json, workspace-mobile.json, graph.json, file-recovery/, cache/, .trash/, *.bak) is gitignored ‚Äî see `.gitignore`.
- `configs/dot-claude-preset/` ‚Äî preset scaffolding **not** wired into the live `~/.claude`: a `session-start-injections/` template (README + settings.json + scafolding) kept as a seed/reference, separate from the `dot-claude` submodule below.
- `configs/dot-claude/` ‚Äî **private git submodule** (`https://github.com/teazyou/dot-claude.git`) that **is** `~/.claude` (Claude Code): on each machine `~/.claude` is a whole-folder **symlink** ‚Üí `configs/dot-claude`, so everything Claude Code writes lands in this submodule. **Self-contained by design:** the submodule alone must deploy the exact same config on a fresh machine ‚Äî everything `~/.claude` needs (settings, scripts like `statusline.sh`, agents, skills, hooks) lives INSIDE it, wired via `~/.claude/...` paths only, never machine-specific (`/Users/<name>/‚Ä¶`) or cross-repo (`~/workspace/‚Ä¶`) paths (philosophy: rule in this repo's `.claude/CLAUDE.md` Conventions + the submodule's README.md). Allowlist-style `.gitignore` (ignore-all + re-allow `settings.json`, `CLAUDE.md`, `keybindings.json`, `statusline.sh`, and the `agents/ commands/ skills/ output-styles/ workflows/ hooks/` dirs; credential denylist) commits only safe config ‚Äî all runtime/secret data (`projects/`, `sessions/`, `history.jsonl`, ‚Ä¶) physically lives here but is gitignored. Secrets are protected three ways: gitignored, private repo, and a mirrored defensive allowlist for `configs/dot-claude/` in this repo's root `.gitignore` (active only if it ever stops being a submodule). *Edit when: changing which `~/.claude` config is backed up or the allowlist; keep the two allowlists in sync. On a fresh Mac this is wired by `scripts/installs/setup_dot_claude.sh`.*
- `configs/dot-claude/statusline.sh` ‚Äî Claude Code status line renderer (text-only context-window usage: model, effort level, colored %, used/total tokens); lives IN the submodule (= `~/.claude/statusline.sh`) so it's versioned in the private dot-claude repo. Invoked by Claude Code via `statusLine.command` in `configs/dot-claude/settings.json` (= `~/.claude/settings.json`, points to `~/.claude/statusline.sh` ‚Äî repo-internal wiring, no machine-specific path); reads the statusline JSON payload on stdin (`.context_window.*` fields), prints one ANSI line. Requires `jq`. *Edit when: changing what the status line shows.*

## zsh
Shell setup; `zshrc.zsh` is the entry point that sources everything else.
- `zsh/zshrc.zsh` ‚Äî `~/.zshrc` source-of-truth; sources every config + alias below ‚Äî `‚Üê ~/.zshrc`. *Edit when: changing what loads at shell startup.*
- `zsh/configs/path.zsh` ‚Äî exports the path vars used everywhere (`WORKSPACE`, `SCRIPTS`, `APP_CONFIGS`, `FUNCTIONS`, `KEYS_DKT`, `ZSH`, `ZSH_CONFIGS`/`ZSH_ALIAS`) and **prepends `~/.local/bin` (`PATH_CLAUDE`, the Claude CLI) to PATH**. *Edit when: adding a new sourced dir or PATH entry.*
- `zsh/configs/*.zsh` ‚Äî startup configs: `colors.zsh`, `oh-my-zsh.zsh` (theme/plugins), `git.zsh` (writes `git config --global` core.editor/user.email/user.name on **every** shell startup), `nvm.zsh`, `iterm2.zsh` (random per-tab colour + `recolor`).
- `zsh/alias/*.zsh` ‚Äî alias groups by topic: `osx.zsh`, `navigation.zsh`, `obsidian.zsh` (the `obsi` vault launcher ‚Üí `scripts/obsi`), `git.zsh`, `installations.zsh`, `checkpoint.zsh` (the `checkpoint` runner ‚Üí `scripts/checkpoint_all.sh`), `claude.zsh` (the `sopaios` launcher ‚Üí `scripts/claude/sopaios.sh`), `vpn.zsh` (the `nord` VPN CLI ‚Üí `scripts/vpn/nord.sh`). **`git.zsh` also defines a `git()` function that reroutes every `git push` through `scripts/git/gpush.sh`** (plain `git push` is intercepted ‚Äî see `docs/scripts/checkpoint-and-git-scripts.md`). *Edit when: adding a terminal alias.*

## scripts
Install, git, system, and checkpoint scripts. The fresh-Mac flow is documented in `docs/install/bootstrap-flow.md`; the checkpoint + git-wrapper systems in `docs/scripts/checkpoint-and-git-scripts.md`.
- `scripts/installs/bootstrap.sh` ‚Äî fresh-Mac entry point (CLT, Homebrew, git, clone repo) then hands off to `installation.sh`. Run remotely via curl one-liner (see file header).
- `scripts/installs/installation.sh` ‚Äî install orchestrator; runs every other `install_*`/`setup_*` step in order (17 numbered steps), idempotently. *Edit when: adding/removing an install step.*
- `scripts/installs/setup_symlinks.sh` ‚Äî creates the repo‚Üísystem symlinks. **NOT the full map: it creates only 6 of the 8 documented links** (it omits the AutoRaise config and the AutoRaise daemon plist, which are wired out-of-band). *Edit when: adding/changing a symlink target ‚Äî and update both this index and `docs/install/bootstrap-flow.md`.*
- `scripts/installs/install_*.sh` ‚Äî one installer per tool/area (brew, node, claude, docling, iterm2, vscode_ext, oh_my_zsh, database, touch_id_sudo, xcode_mas, window_manager, checkpoint_launchd).
- `scripts/installs/setup_macos.sh`, `setup_wallpaper.sh`, `clone_repos.sh` ‚Äî macOS defaults, wallpaper, and repo cloning steps. `clone_repos.sh`'s interactive `gh auth login` is the auth GATE for `setup_dot_claude.sh` / the private submodule.
- `scripts/installs/setup_dot_claude.sh` ‚Äî fresh-machine step (runs after `clone_repos.sh`, once `gh` is authenticated): `gh auth setup-git` ‚Üí `git submodule update --init configs/dot-claude` (the PRIVATE submodule) ‚Üí symlink `~/.claude ‚Üí configs/dot-claude` (backing up any real `~/.claude` to a timestamped `~/.claude.bak.<ts>`, never deleting it). Idempotent. *Edit when: changing how `~/.claude` is wired on a new Mac.*
- `scripts/installs/helper_prompt.sh` ‚Äî `log_*` color/prompt helpers sourced by install scripts.
- `scripts/git/g*.sh` ‚Äî git workflow helpers (`gcommit`, `gcreate`, `gdelete`, `gpush`, `gstatus`); fronted by `zsh/alias/git.zsh`. **`gpush.sh` is the transparent push wrapper** every `git push` routes through (cleans `.DS_Store`, status, real push via `command git push`, then per-repo cleanup + post-push hook). See `docs/scripts/checkpoint-and-git-scripts.md`.
- `scripts/checkpoint_cronjob.sh` ‚Äî LaunchAgent entry: auto-commits/pushes tracked repos, logs to `logs/`. **Inverted gate:** it checkpoints a repo ONLY when its working tree is UNCHANGED since the last run (so in-progress edits are never committed, and an earlier failed push is retried) ‚Äî the opposite of `checkpoint_all.sh`. Invoked by `~/Library/LaunchAgents/com.teazyou.checkpoint.plist`; the **hourly** cadence (minute 0) is defined in the tracked `scripts/installs/install_checkpoint_launchd.sh`.
- `scripts/checkpoint_all.sh`, `checkpoint_functions.sh` ‚Äî manual checkpoint runner (commits **unconditionally**, every repo, every run) + shared logic; `CHECKPOINT_FOLDERS` lists watched repos (`~/workspace/configs/dot-claude` (submodule, listed first so it commits before the parent gitlink), `~/workspace`, `~/secondbrain`). *Edit when: changing which repos auto-checkpoint.*
- `scripts/claude/sopaios.sh` ‚Äî `claude --plugin-dir <SOPAIOS working copy> "$@"` wrapper, so the plugin is loaded from disk (dev mode) instead of a cached install. Fronted by the `sopaios` terminal alias (`zsh/alias/claude.zsh`); plugin dir + claude binary overridable via `SOPAIOS_PLUGIN_DIR`/`SOPAIOS_CLAUDE_BIN`. *Edit when: changing the default plugin working copy or claude binary path.*
- `scripts/claude/quota_keepalive.sh` ‚Äî **experimental** Claude Code quota keepalive: hourly (LaunchAgent above) sends a near-zero-cost "hello" to haiku from `~/tmp` (pseudo-TTY via `script -q`, killed after 15s) so the subscription's 5-hour usage block re-arms continuously ‚Äî after a long absence the current block is already partly elapsed instead of restarting full-length on the first real prompt. Seeds `~/tmp` trust in `~/.claude.json` (jq) on first run; logs to `logs/quota_keepalive.log`. **Not part of the fresh-Mac install flow**; disable via `launchctl bootout` (see script header). *Edit when: changing wait time, model, or work dir.*
- `scripts/vpn/` ‚Äî native NordVPN IKEv2 stack (authoritative guide: `docs/vpn/guide-nordvpn-native.md`): `nord.sh` (the `nord` CLI ‚Äî switch country / on / off / toggle / status / list / refresh / rescue; strict single-slot stop‚Üístart sequencing + shared /tmp lock), `nord-connect.sh` (the LaunchAgent one-shot: boot‚ÜíSingapore reset via kern.bootsessionuuid, event-driven reconnect, no polling, 10-min per-network failure cooldown), `nord-gen-bundle.sh` (renders the 6-country + Root CA `.mobileconfig` from live Nord API best-server data; one manual approval per regeneration). Secrets/state live in `~/.config/nordvpn-native/` (0600), never in the repo. *Edit when: changing countries, reconnect behavior, or profile contents.*
- `scripts/aerospace-restart.sh` ‚Äî full restart of the WM stack (aerospace, sketchybar, borders, and **both** LaunchAgents: display-profile + autoraise).
- `scripts/dstore.sh` ‚Äî recursively delete both `.DS_Store` AND `_DS_Store` files; used by the git scripts (`silent` mode).
- `scripts/obsi` ‚Äî Obsidian vault launcher: opens a folder as a vault, planting per-file symlinks from its `.obsidian/` to the central `configs/dot-obsidian` config. Fronted by `zsh/alias/obsidian.zsh` (`obsi [folder]`). *Edit when: changing which central entries are shared (BLACKLIST) or the open mechanism.*

## functions
Shared SH helpers sourced by other scripts.
- `functions/brew.sh` ‚Äî idempotent `brew` install/tap wrappers used by `scripts/installs/install_brew.sh`. **NOT self-contained:** its `[ OK ]`/`[ W8 ]`/`[ KO ]` log lines use the color vars `COK`/`CW8`/`CKO`/`CWH` defined in `helper_prompt.sh`, which `install_brew.sh` sources immediately before sourcing `brew.sh`.

## Optional / low-touch
- `logs/` ‚Äî runtime output, gitignored except `.gitkeep`: `checkpoint_cron.log`, `checkpoint_launchd.{out,err}.log`, `checkpoint_state/*.sig` (per-repo content fingerprints), `nordvpn-native.log` (VPN reconnect agent), `quota_keepalive.log`.
- `.claude/CLAUDE.md` ‚Äî project instructions for Claude (intended purpose, constraints; points into `docs/`).
- `.gitmodules` ‚Äî git-tracked root file declaring the `configs/dot-claude` submodule (`path = configs/dot-claude`, `url = https://github.com/teazyou/dot-claude.git`).
- `_HUMAN-only.md` ‚Äî a gitignored root note (matched by the `_HUMAN-ONLY.md` rule in `.gitignore`, case-insensitive on APFS). Convention: files matching that rule are **human-only** scratch and are never committed.
- `.gitignore` ‚Äî ignores `.DS_Store`/`._*`/`.vscode`, `*.bak`, `logs/*` (except `.gitkeep`), the `_HUMAN-ONLY.md` human-only rule, and the `.workspace_mysql_secured` install marker. Plus three scoped blocks: the `configs/dot-obsidian/` per-vault runtime state (workspace.json, workspace-mobile.json, graph.json, file-recovery/, cache/, .trash/, *.bak); the sketchybar machine-local artifact (`configs/sketchybar/helpers/wifi_rssi`); and the largest block ‚Äî the **`configs/dot-claude/` defensive allowlist** (ignore-all `configs/dot-claude/*`, re-allow the same safe config the submodule's own `.gitignore` does, then a credential denylist: `settings.local.json`, `**/credentials.json`, `**/.credentials.json`, `**/*.pem`/`*.key`/`*.p12`/`*.pfx`, `**/.env*`, `.claude.json*`). Inert while it's a submodule; only active if it ever stops being one. *(Per-config `.gitignore` files are not used; rules are consolidated here.)*
