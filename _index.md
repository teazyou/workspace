# Repository Index ‚Äî `~/workspace`

> Centralized macOS environment configuration: app configs, shell setup, helper functions, and install/system scripts. This repo is the **source of truth**; most files are symlinked into the locations each app expects (`repo file ‚Üê system path`). See `setup_symlinks.sh` for the symlink wiring (it covers 6 of the 9 documented links ‚Äî the rest are wired out-of-band; see `docs/install/bootstrap-flow.md`).
>
> **KEEP THIS CURRENT:** when you add, move, remove, or rename a file, or change a symlink target, update the matching entry here in the same change.

All prose/context documentation lives in `docs/`, subfoldered by area (`docs/window-manager/`, `docs/vscode/`, `docs/install/`, `docs/scripts/`). Read the relevant guide there before acting on a topic. Guides + reference notes belong in `docs/`; this index + CLAUDE.md point into it. (`prompts/` holds executable launcher prompts, not documentation.)

```
configs/    app configs (aerospace, sketchybar, borders, autoraise, vscode, iterm2)
docs/       prose context guides, subfoldered by area (window-manager, vscode, install, scripts)
zsh/        ~/.zshrc + sourced configs and aliases
scripts/    install/system scripts (installs/, git/, checkpoint, obsi)
functions/  shared SH helpers sourced by scripts
prompts/    reusable copy-paste Claude prompts (workflow launchers)
logs/       runtime logs + checkpoint state (gitignored)
```

## configs
App configuration; the window-manager stack (aerospace + sketchybar + borders + autoraise) is the most-edited area. Its authoritative guide is `docs/window-manager/guide-window-manager.md`.
- `configs/aerospace/aerospace.toml` ‚Äî AeroSpace tiling WM config (workspaces, keybindings, gaps) ‚Äî `‚Üê ~/.aerospace.toml`. Its startup `after-startup-command` sources `lib-paths.sh` before resetting the `/tmp` state files. *Edit when: changing window-manager keybindings/layout. Run `aerospace reload-config` after.*
- `configs/aerospace/*.plist` ‚Äî LaunchAgents: `com.aerospace.display-profile.plist` (`‚Üê ~/Library/LaunchAgents/`, auto gap profile per display) and `com.aerospace.empty-watcher.plist` (empty-workspace daemon ‚Äî **not** symlinked/loaded by the install flow; wired by `aerospace-restart.sh` / by hand).
- `configs/aerospace/*.sh` ‚Äî WM helper scripts run by aerospace/LaunchAgents: `lib-paths.sh` (shared lib, below), `apply-display-profile.sh` (auto top-gaps **and** the workspace 7-9 monitor assignment ‚Äî flips between `'built-in.*'` and `'secondary'` per which display is main, so home/travel both work), `empty-workspace-watcher.sh`, `open-dock-app.sh`, `performance-mode.sh`, `secondary-bar-toggle.sh`, `track-workspace-mru.sh`.
- `configs/aerospace/lib-paths.sh` ‚Äî shared library **`source`d by all six aerospace scripts above AND by aerospace.toml's startup line**: the single source of truth for the `/tmp` state-file paths (`PERFORMANCE_MODE_STATE`, `SECONDARY_BAR_STATE`), the coupled timing constants (`GRACE_SECONDS=20`, `PLACEMENT_CAP_SECONDS=18`, `POLL_INTERVAL=0.5`, `AERO_TIMEOUT=3`), the hang-proof `aero()` wrapper (bash-native `aerospace` watchdog so a wedged server can't permanently hang the watcher), and the path builders (`grace_file`/`mru_file`/`mru_lock`/`file_age_seconds`). *Edit when: changing any cross-script path, timing constant, or the `aero()` call path.*
- `configs/sketchybar/sketchybarrc` ‚Äî status-bar entry point (sources colors/icons/theme, loads items + plugins) ‚Äî dir `‚Üê ~/.config/sketchybar`. *Edit when: changing the status bar.*
- `configs/sketchybar/colors.sh`, `icons.sh` ‚Äî shared palette and icon glyphs. colors.sh's `BORDER_ACTIVE`/`PINK` (`0xffaa2222`) must stay in sync with `configs/borders/bordersrc` (its header says so).
- `configs/sketchybar/theme.sh` ‚Äî **visual template / single source of truth** for "division" geometry (corner radius, border, blur, drop shadow, `GROUP_GAP` between divisions, plus `DIVISION_PAD`/`ELEMENT_GAP` for inner edge + element spacing). Sourced by sketchybarrc before items; every bracket on both sides, in normal + performance mode, pulls from these tokens. *Edit when: restyling the bar's overall pill/division look (radius, shadow, spacing).*
- `configs/sketchybar/helpers/wifi_rssi.swift` ‚Äî tiny CoreWLAN Swift helper printing the current Wi-Fi link RSSI (no scan, no Location permission). plugins/wifi.sh compiles it on demand to the gitignored `helpers/wifi_rssi` binary for the strength icon + weak-signal auto-reconnect (macOS 26 removed `airport`). *Edit when: changing what the helper reports.*
- `configs/sketchybar/helpers/wifi_rssi` ‚Äî the **compiled** Swift binary (gitignored; should exist locally ‚Äî plugins/wifi.sh builds it on first run).
- `configs/sketchybar/wifi_home_ssids.example` ‚Äî template for the gitignored `wifi_home_ssids` (private home SSIDs) that arms wifi.sh's auto-reconnect (toggle Wi-Fi at 1 bar so macOS reconnects to the strongest AP). *Edit when: documenting the auto-reconnect SSID list format.*
- `configs/sketchybar/wifi_home_ssids` ‚Äî the **real** private SSID list (gitignored; should exist locally ‚Äî copy from the `.example`).
- `configs/sketchybar/items/*.sh` ‚Äî one bar item definition per file, BUT not all are wired: only the **12 SOURCED items** are live (spaces, calendar, volume, headset, battery, ram, cpu, vpn, ethernet, wifi, network_down, network_up); `apple.sh`/`settings.sh` are commented and `front_app.sh`/`brew.sh`/`github.sh`/`spotify.sh` are not sourced (6 inert template leftovers). Paddings come from theme.sh tokens. Several live items are state-driven: ethernet/headset show only when connected; volume mutes; vpn = NordVPN glyph; wifi = RSSI bars. **Traffic is TWO independent divisions/brackets** (`traffic_up` + `traffic_down`), each shown only when its own direction clears `MIN_RATE` (~5 KB/s); `network_down` is the SOLE poller (update_freq=5, runs network_speed.sh), `network_up` is passive.
- `configs/sketchybar/plugins/*.sh` ‚Äî update/event scripts backing the items; **the live plugin set is ~15** and ~11 are dead legacy template leftovers. Live + load-bearing: `icon_map.sh` (maps apps to sketchybar-app-font glyphs, used by `aerospace.sh` for the spaces app icons), `aerospace.sh`, `date.sh`, `time.sh`, `wifi.sh` (RSSI strength + auto-reconnect), `network_speed.sh` (VPN-tunnel-safe; SOLE writer of the six traffic items, with a /tmp/performance-mode.state gate + an mkdir lockdir so it never disagrees with performance-mode.sh; hides each idle direction's bracket by toggling BOTH its paint layers ‚Äî `background.drawing` (fill) AND `background.shadow.drawing` (drop shadow) ‚Äî while keeping the bracket `drawing=on`; item-level `drawing=off` stops NEITHER paint layer, it just freezes the bracket wide = a frozen empty pill), plus the per-item updaters (volume, headset, ethernet, ram, cpu, vpn, battery). *Do not assume a plugin is dead just because it's old ‚Äî icon_map/aerospace/date/time are live.*
- `configs/borders/bordersrc` ‚Äî JankyBorders window-border config ‚Äî dir `‚Üê ~/.config/borders`. `active_color=0xffaa2222` MUST stay in sync with colors.sh `BORDER_ACTIVE` (colors.sh's header literally says "keep in sync with configs/borders/bordersrc"). *Edit when: changing border colour/width/style ‚Äî mirror the colour to colors.sh.*
- `configs/autoraise/config` ‚Äî AutoRaise focus-follows-mouse tuning ‚Äî `‚Üê ~/.config/AutoRaise/config` (also lowercase `autoraise/`). The live config sets `requireMouseStop=false` (no rest/dwell model ‚Äî the coarse `pollMillis` IS the fly-over debounce) and `ignoreSpaceChanged=true`. *Edit when: tuning poll timing / fly-over debounce.*
- `configs/autoraise/com.autoraise.daemon.plist` ‚Äî AutoRaise LaunchAgent ‚Äî `‚Üê ~/Library/LaunchAgents/` (note: AutoRaise's binary, config symlink, and daemon are **not** installed/wired by the fresh-Mac flow; see `docs/install/bootstrap-flow.md`).
- `configs/vscode/settings.json` ‚Äî VS Code user settings ‚Äî `‚Üê ~/Library/Application Support/Code/User/settings.json`. *Read `docs/vscode/transparency.md` before editing theme/colour/transparency settings.*
- `configs/vscode/custom.css` ‚Äî custom workbench CSS injected by Vibrancy Continued via `vscode_vibrancy.imports` in settings.json (no Custom-CSS extension needed). Used for things VS Code has no native setting for ‚Äî currently a **2px active-tab accent on the native `.tab-border-bottom-container` element** (focused/unfocused editor-group split). *Edit when: tweaking the injected CSS; run "Reload Vibrancy" after. Documented in `docs/vscode/transparency.md`.*
- `configs/iterm2/com.googlecode.iterm2.plist` ‚Äî iTerm2 prefs. Loaded via iTerm2's **"Load preferences from custom folder"** (a two-way LIVE file at `$APP_CONFIGS/iterm2` via `PrefsCustomFolder` + `LoadPrefsFromCustomFolder`, **NOT a symlink**), wired by `scripts/installs/install_iterm2.sh`. **Gotcha: iTerm2 must NOT be running when those defaults are written or it overwrites the plist on quit.** (The `recolor` random-tab-color helper lives in `zsh/configs/iterm2.zsh`.)
- `configs/dot-obsidian/` ‚Äî central Obsidian config (plugins, snippets, hotkeys, appearance, *.json settings). Not symlinked as a whole; the `obsi` launcher (`scripts/obsi`) plants per-file symlinks from each vault's `.obsidian/` to entries here (the per-file-symlink/BLACKLIST model is documented in `scripts/obsi`). Per-vault runtime state (workspace.json, workspace-mobile.json, graph.json, file-recovery/, cache/, .trash/, *.bak) is gitignored ‚Äî see `.gitignore`.
- `configs/dot-claude-preset/` ‚Äî preset scaffolding **not** wired into the live `~/.claude`: a `session-start-injections/` template (README + settings.json + scafolding) kept as a seed/reference, separate from the `dot-claude` submodule below.
- `configs/dot-claude/` ‚Äî **private git submodule** (`https://github.com/teazyou/dot-claude.git`) that **is** `~/.claude` (Claude Code): on each machine `~/.claude` is a whole-folder **symlink** ‚Üí `configs/dot-claude`, so everything Claude Code writes lands in this submodule. Allowlist-style `.gitignore` (ignore-all + re-allow `settings.json`, `CLAUDE.md`, `keybindings.json`, and the `agents/ commands/ skills/ output-styles/ workflows/ hooks/` dirs; credential denylist) commits only safe config ‚Äî all runtime/secret data (`projects/`, `sessions/`, `history.jsonl`, ‚Ä¶) physically lives here but is gitignored. Secrets are protected three ways: gitignored, private repo, and a mirrored defensive allowlist for `configs/dot-claude/` in this repo's root `.gitignore` (active only if it ever stops being a submodule). *Edit when: changing which `~/.claude` config is backed up or the allowlist; keep the two allowlists in sync. On a fresh Mac this is wired by `scripts/installs/setup_dot_claude.sh`.*

## zsh
Shell setup; `zshrc.zsh` is the entry point that sources everything else.
- `zsh/zshrc.zsh` ‚Äî `~/.zshrc` source-of-truth; sources every config + alias below ‚Äî `‚Üê ~/.zshrc`. *Edit when: changing what loads at shell startup.*
- `zsh/configs/path.zsh` ‚Äî exports the path vars used everywhere (`WORKSPACE`, `SCRIPTS`, `APP_CONFIGS`, `FUNCTIONS`, `KEYS_DKT`, `ZSH`, `ZSH_CONFIGS`/`ZSH_ALIAS`) and **prepends `~/.local/bin` (`PATH_CLAUDE`, the Claude CLI) to PATH**. *Edit when: adding a new sourced dir or PATH entry.*
- `zsh/configs/*.zsh` ‚Äî startup configs: `colors.zsh`, `oh-my-zsh.zsh` (theme/plugins), `git.zsh` (writes `git config --global` core.editor/user.email/user.name on **every** shell startup), `nvm.zsh`, `iterm2.zsh` (random per-tab colour + `recolor`).
- `zsh/alias/*.zsh` ‚Äî alias groups by topic: `osx.zsh`, `navigation.zsh`, `obsidian.zsh` (the `obsi` vault launcher ‚Üí `scripts/obsi`), `git.zsh`, `installations.zsh`, `checkpoint.zsh` (the `checkpoint` runner ‚Üí `scripts/checkpoint_all.sh`), `wallpapers.zsh` (the `wallpapers-treatment` launcher ‚Üí `scripts/wallpapers_treatment.sh`), `claude.zsh` (the `sopaios` launcher ‚Üí `scripts/claude/sopaios.sh`). **`git.zsh` also defines a `git()` function that reroutes every `git push` through `scripts/git/gpush.sh`** (plain `git push` is intercepted ‚Äî see `docs/scripts/checkpoint-and-git-scripts.md`). *Edit when: adding a terminal alias.*

## scripts
Install, git, system, and checkpoint scripts. The fresh-Mac flow is documented in `docs/install/bootstrap-flow.md`; the checkpoint + git-wrapper systems in `docs/scripts/checkpoint-and-git-scripts.md`.
- `scripts/installs/bootstrap.sh` ‚Äî fresh-Mac entry point (CLT, Homebrew, git, clone repo) then hands off to `installation.sh`. Run remotely via curl one-liner (see file header).
- `scripts/installs/installation.sh` ‚Äî install orchestrator; runs every other `install_*`/`setup_*` step in order (17 numbered steps), idempotently. *Edit when: adding/removing an install step.*
- `scripts/installs/setup_symlinks.sh` ‚Äî creates the repo‚Üísystem symlinks. **NOT the full map: it creates only 6 of the 9 documented links** (it omits the AutoRaise config, the AutoRaise daemon plist, and the aerospace empty-watcher plist, which are wired out-of-band). *Edit when: adding/changing a symlink target ‚Äî and update both this index and `docs/install/bootstrap-flow.md`.*
- `scripts/installs/install_*.sh` ‚Äî one installer per tool/area (brew, node, claude, docling, iterm2, vscode_ext, oh_my_zsh, database, touch_id_sudo, xcode_mas, window_manager, checkpoint_launchd).
- `scripts/installs/setup_macos.sh`, `setup_wallpaper.sh`, `clone_repos.sh` ‚Äî macOS defaults, wallpaper, and repo cloning steps. `clone_repos.sh`'s interactive `gh auth login` is the auth GATE for `setup_dot_claude.sh` / the private submodule.
- `scripts/installs/setup_dot_claude.sh` ‚Äî fresh-machine step (runs after `clone_repos.sh`, once `gh` is authenticated): `gh auth setup-git` ‚Üí `git submodule update --init configs/dot-claude` (the PRIVATE submodule) ‚Üí symlink `~/.claude ‚Üí configs/dot-claude` (backing up any real `~/.claude` to a timestamped `~/.claude.bak.<ts>`, never deleting it). Idempotent. *Edit when: changing how `~/.claude` is wired on a new Mac.*
- `scripts/installs/helper_prompt.sh` ‚Äî `log_*` color/prompt helpers sourced by install scripts.
- `scripts/git/g*.sh` ‚Äî git workflow helpers (`gcommit`, `gcreate`, `gdelete`, `gpush`, `gstatus`); fronted by `zsh/alias/git.zsh`. **`gpush.sh` is the transparent push wrapper** every `git push` routes through (cleans `.DS_Store`, status, real push via `command git push`, then per-repo cleanup + post-push hook). See `docs/scripts/checkpoint-and-git-scripts.md`.
- `scripts/checkpoint_cronjob.sh` ‚Äî LaunchAgent entry: auto-commits/pushes tracked repos, logs to `logs/`. **Inverted gate:** it checkpoints a repo ONLY when its working tree is UNCHANGED since the last run (so in-progress edits are never committed, and an earlier failed push is retried) ‚Äî the opposite of `checkpoint_all.sh`. Invoked by `~/Library/LaunchAgents/com.teazyou.checkpoint.plist`; the **hourly** cadence (minute 0) is defined in the tracked `scripts/installs/install_checkpoint_launchd.sh`.
- `scripts/checkpoint_all.sh`, `checkpoint_functions.sh` ‚Äî manual checkpoint runner (commits **unconditionally**, every repo, every run) + shared logic; `CHECKPOINT_FOLDERS` lists watched repos (`~/workspace/configs/dot-claude` (submodule, listed first so it commits before the parent gitlink), `~/workspace`, `~/secondbrain`). *Edit when: changing which repos auto-checkpoint.*
- `scripts/claude/sopaios.sh` ‚Äî `claude --plugin-dir <SOPAIOS working copy> "$@"` wrapper, so the plugin is loaded from disk (dev mode) instead of a cached install. Two consumers: the `sopaios` terminal alias (`zsh/alias/claude.zsh`), and the VS Code extension's `claudeCode.claudeProcessWrapper` setting ‚Äî which takes a **path, not a command line**, and (on this install, where the extension bundles no CLI so `executableArgs` is `[]`) is spawned directly as the claude executable, letting the script inject the flag. Absolute `claude` path required (the extension host's PATH ‚âÝ the shell's); both paths overridable via `SOPAIOS_PLUGIN_DIR`/`SOPAIOS_CLAUDE_BIN`; guards against a future bundled-CLI `wrapper <node> <cli.js> ‚Ä¶` invocation. *Read `docs/vscode/claude-plugin-dev-wrapper.md` before editing; VS Code needs a window reload after a setting change.*
- `scripts/aerospace-restart.sh` ‚Äî full restart of the WM stack (aerospace, sketchybar, borders, and **all three** LaunchAgents incl. autoraise + empty-watcher).
- `scripts/wallpapers_treatment.sh` ‚Äî batch-applies an ImageMagick "profile" (e.g. `blur-4`) to every wallpaper under `~/gdrive/wallpapers/originals/<category>/`, writing processed copies into `~/gdrive/wallpapers/modified/<profile-name>/<profile-name>-<category>/` (originals never modified; idempotent ‚Äî skips already-processed images). `originals/` is the source of truth: a clean pass first prunes `modified/` to mirror it (removes any profile subfolder/image whose original is gone, even for profiles no longer in the array). Failed conversions are retried up to 3 rounds. Self-installs ImageMagick via Homebrew if missing. Profiles are an array at the top of the file (`"<name>|<magick options>"`); fronted by `zsh/alias/wallpapers.zsh` (`wallpapers-treatment <profile-name>`). Bash 3.2-compatible. *Edit when: adding/changing a profile.*
- `scripts/dstore.sh` ‚Äî recursively delete both `.DS_Store` AND `_DS_Store` files; used by the git scripts (`silent` mode).
- `scripts/obsi` ‚Äî Obsidian vault launcher: opens a folder as a vault, planting per-file symlinks from its `.obsidian/` to the central `configs/dot-obsidian` config. Fronted by `zsh/alias/obsidian.zsh` (`obsi [folder]`). *Edit when: changing which central entries are shared (BLACKLIST) or the open mechanism.*

## prompts
Reusable copy-paste Claude prompts; not sourced by anything ‚Äî saved so a task can be re-launched verbatim in a fresh session.
- `prompts/workflow/window-manager-code-review.md` ‚Äî paste-prompt that launches the multi-agent dynamic Workflow (study ‚Üí review ‚Üí plan ‚Üí coherence ‚Üí implement ‚Üí verify ‚Üí report) over the window-manager config stack. Its first step is a **study** agent (Opus, xhigh) that reads `docs/window-manager/guide-window-manager.md` (and the config folders) and returns the config-unit and lens lists (also written to `_spec/workflow-spec.json` as an artifact); the Workflow then fans out the fixed split strategy (one sub-agent per config-unit √ó lens) over those lists at runtime ‚Äî one self-contained autonomous run. All artifacts go to a freshly-recreated `./window-manager-code-review/` at the repo root. *Edit when: changing the review pipeline's steps/units/lenses.*

## Optional / low-touch
- `logs/` ‚Äî runtime output, gitignored except `.gitkeep`: `checkpoint_cron.log`, `checkpoint_launchd.{out,err}.log`, `checkpoint_state/*.sig` (per-repo content fingerprints).
- `.claude/CLAUDE.md` ‚Äî project instructions for Claude (intended purpose, constraints; points into `docs/`).
- `.gitmodules` ‚Äî git-tracked root file declaring the `configs/dot-claude` submodule (`path = configs/dot-claude`, `url = https://github.com/teazyou/dot-claude.git`).
- `_HUMAN-only.md` ‚Äî a gitignored root note (matched by the `_HUMAN-ONLY.md` rule in `.gitignore`, case-insensitive on APFS). Convention: files matching that rule are **human-only** scratch and are never committed.
- `.gitignore` ‚Äî ignores `.DS_Store`/`._*`/`.vscode`, `*.bak`, `logs/*` (except `.gitkeep`), the `_HUMAN-ONLY.md` human-only rule, and the `.workspace_mysql_secured` install marker. Plus three scoped blocks: the `configs/dot-obsidian/` per-vault runtime state (workspace.json, workspace-mobile.json, graph.json, file-recovery/, cache/, .trash/, *.bak); the sketchybar machine-local artifacts (`configs/sketchybar/helpers/wifi_rssi`, `configs/sketchybar/wifi_home_ssids`); and the largest block ‚Äî the **`configs/dot-claude/` defensive allowlist** (ignore-all `configs/dot-claude/*`, re-allow the same safe config the submodule's own `.gitignore` does, then a credential denylist: `settings.local.json`, `**/credentials.json`, `**/.credentials.json`, `**/*.pem`/`*.key`/`*.p12`/`*.pfx`, `**/.env*`, `.claude.json*`). Inert while it's a submodule; only active if it ever stops being one. *(Per-config `.gitignore` files are not used; rules are consolidated here.)*
