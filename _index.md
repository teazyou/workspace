# Repository Index — `~/workspace`

> Centralized macOS environment configuration: app configs, shell setup, helper functions, and install/system scripts. This repo is the **source of truth**; most files are symlinked into the locations each app expects (`repo file ← system path`). See `setup_symlinks.sh` for the symlink wiring (it covers 6 of the 9 documented links — the rest are wired out-of-band; see `docs/install/bootstrap-flow.md`).
>
> **KEEP THIS CURRENT:** when you add, move, remove, or rename a file, or change a symlink target, update the matching entry here in the same change.

All prose/context documentation lives in `docs/`, subfoldered by area (`docs/window-manager/`, `docs/install/`, `docs/scripts/`). Read the relevant guide there before acting on a topic. Guides + reference notes belong in `docs/`; this index + CLAUDE.md point into it. (`prompts/` holds executable launcher prompts, not documentation.)

```
configs/    app configs (aerospace, sketchybar, borders, autoraise, vscode, iterm2)
docs/       prose context guides, subfoldered by area (window-manager, install, scripts)
zsh/        ~/.zshrc + sourced configs and aliases
scripts/    install/system scripts (installs/, git/, checkpoint, obsi)
functions/  shared SH helpers sourced by scripts
prompts/    reusable copy-paste Claude prompts (workflow launchers)
logs/       runtime logs + checkpoint state (gitignored)
```

## docs
Prose context guides, subfoldered by area. Read the relevant one before acting on its topic; each is the authoritative extended context for that area (config files stay the runtime source of truth).
- `docs/window-manager/guide-window-manager.md` — **authoritative** window-manager guide (AeroSpace + SketchyBar + JankyBorders + AutoRaise): the symlink map, every config/script with its rationale, and the cross-script /tmp + timing contract. *Read before/after editing the WM stack.*
- `docs/window-manager/doc.aerospace.md` — GENERIC upstream AeroSpace/SketchyBar/JankyBorders install tutorial. **NON-AUTHORITATIVE:** uses default/example values that do NOT match this repo's live config. Reference only.
- `docs/window-manager/features.aerospace.md` — GENERIC upstream AeroSpace feature/keybinding reference. **NON-AUTHORITATIVE:** lists keybindings (e.g. `alt+f`, service-mode `backspace`) absent from the live config. Reference only.
- `docs/install/bootstrap-flow.md` — fresh-Mac install & bootstrap orchestration: the two-stage `bootstrap.sh`→`installation.sh` handoff, the 17-step ordering + dependency constraints, per-step idempotency checks, and the gaps the flow does NOT wire (AutoRaise binary/config/daemon). *Read for any fresh-Mac setup or `scripts/installs/` change.*
- `docs/scripts/checkpoint-and-git-scripts.md` — the checkpoint auto-commit/auto-push system (the inverted hourly gate vs the unconditional manual runner) and the transparent `git push` wrapper (the zsh `git()` function that reroutes every push through `gpush.sh`). *Read when touching `scripts/git/` or the `checkpoint_*` scripts.*
- `docs/scripts/quota-keepalive.md` — the **experimental** Claude quota-keepalive LaunchAgent: how the hourly `haiku` "hello" (pseudo-TTY via `script -q`, killed after 15s, from `~/tmp`) keeps the subscription's 5-hour usage block re-arming, where the script/plist/logs live, the first-run `~/.claude.json` trust seeding, and the by-hand `launchctl bootstrap`/`bootout` enable/disable (deliberately NOT in the fresh-Mac install flow). *Read when touching `scripts/claude/quota_keepalive.sh` or its plist.*

## configs
App configuration; the window-manager stack (aerospace + sketchybar + borders + autoraise) is the most-edited area. Its authoritative guide is `docs/window-manager/guide-window-manager.md`.
- `configs/aerospace/aerospace.toml` — AeroSpace tiling WM config (workspaces, keybindings, gaps) — `← ~/.aerospace.toml`. Its startup `after-startup-command` sources `lib-paths.sh` before resetting the `/tmp` secondary-bar state file (bar defaults to hidden on secondary monitors). *Edit when: changing window-manager keybindings/layout. Run `aerospace reload-config` after.*
- `configs/aerospace/*.plist` — LaunchAgent: `com.aerospace.display-profile.plist` (`← ~/Library/LaunchAgents/`, auto gap profile per display, StartInterval 60).
- `configs/aerospace/*.sh` — WM helper scripts run by aerospace/LaunchAgents: `lib-paths.sh` (shared lib, below), `apply-display-profile.sh` (auto top-gaps **and** the workspace 7-9 monitor assignment — flips between `'built-in.*'` and `'secondary'` per which display is main, so home/travel both work), `open-dock-app.sh`, `secondary-bar-toggle.sh`.
- `configs/aerospace/lib-paths.sh` — shared library **`source`d by the aerospace scripts above AND by aerospace.toml's startup line**: the single source of truth for the `/tmp` state-file path (`SECONDARY_BAR_STATE`) and `PLACEMENT_CAP_SECONDS=18` (open-dock-app's placement-enforcer cap). *Edit when: changing any cross-script path or timing constant.* (The empty-workspace watcher/MRU/grace/`aero()` stack was removed 2026-07 — AeroSpace natively reveals the next focused window's workspace when the last window closes.)
- `configs/sketchybar/sketchybarrc` — status-bar entry point (sources colors/icons/theme, loads items + plugins) — dir `← ~/.config/sketchybar`. *Edit when: changing the status bar.*
- `configs/sketchybar/colors.sh`, `icons.sh` — shared palette and icon glyphs. colors.sh's `BORDER_ACTIVE`/`PINK` (`0xffaa2222`) must stay in sync with `configs/borders/bordersrc` (its header says so).
- `configs/sketchybar/theme.sh` — **visual template / single source of truth** for "division" geometry (corner radius, border, blur, drop shadow, `GROUP_GAP` between divisions, plus `DIVISION_PAD`/`ELEMENT_GAP` for inner edge + element spacing). Sourced by sketchybarrc before items; every bracket on both sides pulls from these tokens. *Edit when: restyling the bar's overall pill/division look (radius, shadow, spacing).*
- `configs/sketchybar/helpers/wifi_rssi.swift` — tiny CoreWLAN Swift helper printing the current Wi-Fi link RSSI (no scan, no Location permission). plugins/wifi.sh compiles it on demand to the gitignored `helpers/wifi_rssi` binary for the strength icon (macOS 26 removed `airport`). *Edit when: changing what the helper reports.*
- `configs/sketchybar/helpers/wifi_rssi` — the **compiled** Swift binary (gitignored; should exist locally — plugins/wifi.sh builds it on first run).
- `configs/sketchybar/items/*.sh` — one bar item definition per file, BUT not all are wired: only the **10 SOURCED items** are live (spaces, calendar, volume, headset, battery, ram, cpu, vpn, ethernet, wifi); `apple.sh`/`settings.sh` are commented and `front_app.sh`/`brew.sh`/`github.sh`/`spotify.sh` are not sourced (6 inert template leftovers). Paddings come from theme.sh tokens. Several live items are state-driven: ethernet/headset show only when connected; volume mutes; vpn = NordVPN glyph; wifi = RSSI bars.
- `configs/sketchybar/plugins/*.sh` — update/event scripts backing the items; **the live plugin set is ~14** and ~11 are dead legacy template leftovers. Live + load-bearing: `icon_map.sh` (maps apps to sketchybar-app-font glyphs, used by `aerospace.sh` for the spaces app icons), `aerospace.sh`, `date.sh`, `time.sh`, `wifi.sh` (RSSI strength icon), plus the per-item updaters (volume, headset, ethernet, ram, cpu, vpn, battery). *Do not assume a plugin is dead just because it's old — icon_map/aerospace/date/time are live.*
- `configs/borders/bordersrc` — JankyBorders window-border config — dir `← ~/.config/borders`. `active_color=0xffaa2222` MUST stay in sync with colors.sh `BORDER_ACTIVE` (colors.sh's header literally says "keep in sync with configs/borders/bordersrc"). *Edit when: changing border colour/width/style — mirror the colour to colors.sh.*
- `configs/autoraise/config` — AutoRaise focus-follows-mouse tuning — `← ~/.config/AutoRaise/config` (also lowercase `autoraise/`). The live config sets `requireMouseStop=false` (no rest/dwell model — the coarse `pollMillis` IS the fly-over debounce) and `ignoreSpaceChanged=true`. *Edit when: tuning poll timing / fly-over debounce.*
- `configs/autoraise/com.autoraise.daemon.plist` — AutoRaise LaunchAgent — `← ~/Library/LaunchAgents/` (note: AutoRaise's binary, config symlink, and daemon are **not** installed/wired by the fresh-Mac flow; see `docs/install/bootstrap-flow.md`).
- `configs/vscode/settings.json` — VS Code user settings — `← ~/Library/Application Support/Code/User/settings.json`.
- `configs/iterm2/com.googlecode.iterm2.plist` — iTerm2 prefs. Loaded via iTerm2's **"Load preferences from custom folder"** (a two-way LIVE file at `$APP_CONFIGS/iterm2` via `PrefsCustomFolder` + `LoadPrefsFromCustomFolder`, **NOT a symlink**), wired by `scripts/installs/install_iterm2.sh`. **Gotcha: iTerm2 must NOT be running when those defaults are written or it overwrites the plist on quit.** (The `recolor` random-tab-color helper lives in `zsh/configs/iterm2.zsh`.)
- `configs/claude/com.teazyou.claude-quota-keepalive.plist` — **experimental** LaunchAgent (`← ~/Library/LaunchAgents/`, StartInterval 3600) firing `scripts/claude/quota_keepalive.sh` hourly. **Deliberately NOT wired into the install flow** (installation.sh / setup_symlinks.sh) — loaded by hand via `launchctl bootstrap`; disable/remove instructions in the script header. *Edit when: changing the ping cadence.*
- `configs/dot-obsidian/` — central Obsidian config (plugins, snippets, hotkeys, appearance, *.json settings). Not symlinked as a whole; the `obsi` launcher (`scripts/obsi`) plants per-file symlinks from each vault's `.obsidian/` to entries here (the per-file-symlink/BLACKLIST model is documented in `scripts/obsi`). Per-vault runtime state (workspace.json, workspace-mobile.json, graph.json, file-recovery/, cache/, .trash/, *.bak) is gitignored — see `.gitignore`.
- `configs/dot-claude-preset/` — preset scaffolding **not** wired into the live `~/.claude`: a `session-start-injections/` template (README + settings.json + scafolding) kept as a seed/reference, separate from the `dot-claude` submodule below.
- `configs/dot-claude/` — **private git submodule** (`https://github.com/teazyou/dot-claude.git`) that **is** `~/.claude` (Claude Code): on each machine `~/.claude` is a whole-folder **symlink** → `configs/dot-claude`, so everything Claude Code writes lands in this submodule. Allowlist-style `.gitignore` (ignore-all + re-allow `settings.json`, `CLAUDE.md`, `keybindings.json`, and the `agents/ commands/ skills/ output-styles/ workflows/ hooks/` dirs; credential denylist) commits only safe config — all runtime/secret data (`projects/`, `sessions/`, `history.jsonl`, …) physically lives here but is gitignored. Secrets are protected three ways: gitignored, private repo, and a mirrored defensive allowlist for `configs/dot-claude/` in this repo's root `.gitignore` (active only if it ever stops being a submodule). *Edit when: changing which `~/.claude` config is backed up or the allowlist; keep the two allowlists in sync. On a fresh Mac this is wired by `scripts/installs/setup_dot_claude.sh`.*

## zsh
Shell setup; `zshrc.zsh` is the entry point that sources everything else.
- `zsh/zshrc.zsh` — `~/.zshrc` source-of-truth; sources every config + alias below — `← ~/.zshrc`. *Edit when: changing what loads at shell startup.*
- `zsh/configs/path.zsh` — exports the path vars used everywhere (`WORKSPACE`, `SCRIPTS`, `APP_CONFIGS`, `FUNCTIONS`, `KEYS_DKT`, `ZSH`, `ZSH_CONFIGS`/`ZSH_ALIAS`) and **prepends `~/.local/bin` (`PATH_CLAUDE`, the Claude CLI) to PATH**. *Edit when: adding a new sourced dir or PATH entry.*
- `zsh/configs/*.zsh` — startup configs: `colors.zsh`, `oh-my-zsh.zsh` (theme/plugins), `git.zsh` (writes `git config --global` core.editor/user.email/user.name on **every** shell startup), `nvm.zsh`, `iterm2.zsh` (random per-tab colour + `recolor`).
- `zsh/alias/*.zsh` — alias groups by topic: `osx.zsh`, `navigation.zsh`, `obsidian.zsh` (the `obsi` vault launcher → `scripts/obsi`), `git.zsh`, `installations.zsh`, `checkpoint.zsh` (the `checkpoint` runner → `scripts/checkpoint_all.sh`), `wallpapers.zsh` (the `wallpapers-treatment` launcher → `scripts/wallpapers_treatment.sh`), `claude.zsh` (the `sopaios` launcher → `scripts/claude/sopaios.sh`). **`git.zsh` also defines a `git()` function that reroutes every `git push` through `scripts/git/gpush.sh`** (plain `git push` is intercepted — see `docs/scripts/checkpoint-and-git-scripts.md`). *Edit when: adding a terminal alias.*

## scripts
Install, git, system, and checkpoint scripts. The fresh-Mac flow is documented in `docs/install/bootstrap-flow.md`; the checkpoint + git-wrapper systems in `docs/scripts/checkpoint-and-git-scripts.md`.
- `scripts/installs/bootstrap.sh` — fresh-Mac entry point (CLT, Homebrew, git, clone repo) then hands off to `installation.sh`. Run remotely via curl one-liner (see file header).
- `scripts/installs/installation.sh` — install orchestrator; runs every other `install_*`/`setup_*` step in order (17 numbered steps), idempotently. *Edit when: adding/removing an install step.*
- `scripts/installs/setup_symlinks.sh` — creates the repo→system symlinks. **NOT the full map: it creates only 6 of the 8 documented links** (it omits the AutoRaise config and the AutoRaise daemon plist, which are wired out-of-band). *Edit when: adding/changing a symlink target — and update both this index and `docs/install/bootstrap-flow.md`.*
- `scripts/installs/install_*.sh` — one installer per tool/area (brew, node, claude, docling, iterm2, vscode_ext, oh_my_zsh, database, touch_id_sudo, xcode_mas, window_manager, checkpoint_launchd).
- `scripts/installs/setup_macos.sh`, `setup_wallpaper.sh`, `clone_repos.sh` — macOS defaults, wallpaper, and repo cloning steps. `clone_repos.sh`'s interactive `gh auth login` is the auth GATE for `setup_dot_claude.sh` / the private submodule.
- `scripts/installs/setup_dot_claude.sh` — fresh-machine step (runs after `clone_repos.sh`, once `gh` is authenticated): `gh auth setup-git` → `git submodule update --init configs/dot-claude` (the PRIVATE submodule) → symlink `~/.claude → configs/dot-claude` (backing up any real `~/.claude` to a timestamped `~/.claude.bak.<ts>`, never deleting it). Idempotent. *Edit when: changing how `~/.claude` is wired on a new Mac.*
- `scripts/installs/helper_prompt.sh` — `log_*` color/prompt helpers sourced by install scripts.
- `scripts/git/g*.sh` — git workflow helpers (`gcommit`, `gcreate`, `gdelete`, `gpush`, `gstatus`); fronted by `zsh/alias/git.zsh`. **`gpush.sh` is the transparent push wrapper** every `git push` routes through (cleans `.DS_Store`, status, real push via `command git push`, then per-repo cleanup + post-push hook). See `docs/scripts/checkpoint-and-git-scripts.md`.
- `scripts/checkpoint_cronjob.sh` — LaunchAgent entry: auto-commits/pushes tracked repos, logs to `logs/`. **Inverted gate:** it checkpoints a repo ONLY when its working tree is UNCHANGED since the last run (so in-progress edits are never committed, and an earlier failed push is retried) — the opposite of `checkpoint_all.sh`. Invoked by `~/Library/LaunchAgents/com.teazyou.checkpoint.plist`; the **hourly** cadence (minute 0) is defined in the tracked `scripts/installs/install_checkpoint_launchd.sh`.
- `scripts/checkpoint_all.sh`, `checkpoint_functions.sh` — manual checkpoint runner (commits **unconditionally**, every repo, every run) + shared logic; `CHECKPOINT_FOLDERS` lists watched repos (`~/workspace/configs/dot-claude` (submodule, listed first so it commits before the parent gitlink), `~/workspace`, `~/secondbrain`). *Edit when: changing which repos auto-checkpoint.*
- `scripts/claude/sopaios.sh` — `claude --plugin-dir <SOPAIOS working copy> "$@"` wrapper, so the plugin is loaded from disk (dev mode) instead of a cached install. Fronted by the `sopaios` terminal alias (`zsh/alias/claude.zsh`); plugin dir + claude binary overridable via `SOPAIOS_PLUGIN_DIR`/`SOPAIOS_CLAUDE_BIN`. *Edit when: changing the default plugin working copy or claude binary path.*
- `scripts/claude/statusline.sh` — Claude Code status line renderer (text-only context-window usage: model, effort level, colored %, used/total tokens). Invoked by Claude Code via `statusLine.command` in `configs/dot-claude/settings.json` (= `~/.claude/settings.json`); reads the statusline JSON payload on stdin (`.context_window.*` fields), prints one ANSI line. Requires `jq`. *Edit when: changing what the status line shows.*
- `scripts/claude/quota_keepalive.sh` — **experimental** Claude Code quota keepalive: hourly (LaunchAgent above) sends a near-zero-cost "hello" to haiku from `~/tmp` (pseudo-TTY via `script -q`, killed after 15s) so the subscription's 5-hour usage block re-arms continuously — after a long absence the current block is already partly elapsed instead of restarting full-length on the first real prompt. Seeds `~/tmp` trust in `~/.claude.json` (jq) on first run; logs to `logs/quota_keepalive.log`. **Not part of the fresh-Mac install flow**; disable via `launchctl bootout` (see script header). *Edit when: changing wait time, model, or work dir.*
- `scripts/aerospace-restart.sh` — full restart of the WM stack (aerospace, sketchybar, borders, and **both** LaunchAgents: display-profile + autoraise).
- `scripts/wallpapers_treatment.sh` — batch-applies an ImageMagick "profile" (e.g. `blur-4`) to every wallpaper under `~/gdrive/wallpapers/originals/<category>/`, writing processed copies into `~/gdrive/wallpapers/modified/<profile-name>/<profile-name>-<category>/` (originals never modified; idempotent — skips already-processed images). `originals/` is the source of truth: a clean pass first prunes `modified/` to mirror it (removes any profile subfolder/image whose original is gone, even for profiles no longer in the array). Failed conversions are retried up to 3 rounds. Self-installs ImageMagick via Homebrew if missing. Profiles are an array at the top of the file (`"<name>|<magick options>"`); fronted by `zsh/alias/wallpapers.zsh` (`wallpapers-treatment <profile-name>`). Bash 3.2-compatible. *Edit when: adding/changing a profile.*
- `scripts/dstore.sh` — recursively delete both `.DS_Store` AND `_DS_Store` files; used by the git scripts (`silent` mode).
- `scripts/obsi` — Obsidian vault launcher: opens a folder as a vault, planting per-file symlinks from its `.obsidian/` to the central `configs/dot-obsidian` config. Fronted by `zsh/alias/obsidian.zsh` (`obsi [folder]`). *Edit when: changing which central entries are shared (BLACKLIST) or the open mechanism.*

## functions
Shared SH helpers sourced by other scripts.
- `functions/brew.sh` — idempotent `brew` install/tap wrappers used by `scripts/installs/install_brew.sh`. **NOT self-contained:** its `[ OK ]`/`[ W8 ]`/`[ KO ]` log lines use the color vars `COK`/`CW8`/`CKO`/`CWH` defined in `helper_prompt.sh`, which `install_brew.sh` sources immediately before sourcing `brew.sh`.

## prompts
Reusable copy-paste Claude prompts; not sourced by anything — saved so a task can be re-launched verbatim in a fresh session.
- `prompts/workflow/window-manager-code-review.md` — paste-prompt that launches the multi-agent dynamic Workflow (study → review → plan → coherence → implement → verify → report) over the window-manager config stack. Its first step is a **study** agent (Opus, xhigh) that reads `docs/window-manager/guide-window-manager.md` (and the config folders) and returns the config-unit and lens lists (also written to `_spec/workflow-spec.json` as an artifact); the Workflow then fans out the fixed split strategy (one sub-agent per config-unit × lens) over those lists at runtime — one self-contained autonomous run. All artifacts go to a freshly-recreated `./window-manager-code-review/` at the repo root. *Edit when: changing the review pipeline's steps/units/lenses.*
- `prompts/claude/caveman-compress-test-and-fix.md` — paste-prompt: e2e-test `scripts/claude/caveman_compress.sh` (cheap `--haiku --low` run on an out-of-repo scratch file) and fix it if broken, with the spec invariants a fix must preserve (no `-p`, 300ms poll, kill-own-session-only, defaults), the failure-suspect list (trust dialog, missing slash command, TUI line-wrap vs `</result>`, grace too short, binary path), and the commit rules (submodule first, pathspec-only commits). *Edit when: the script's spec/defaults change.*

## Optional / low-touch
- `logs/` — runtime output, gitignored except `.gitkeep`: `checkpoint_cron.log`, `checkpoint_launchd.{out,err}.log`, `checkpoint_state/*.sig` (per-repo content fingerprints).
- `.claude/CLAUDE.md` — project instructions for Claude (intended purpose, constraints; points into `docs/`).
- `.gitmodules` — git-tracked root file declaring the `configs/dot-claude` submodule (`path = configs/dot-claude`, `url = https://github.com/teazyou/dot-claude.git`).
- `_HUMAN-only.md` — a gitignored root note (matched by the `_HUMAN-ONLY.md` rule in `.gitignore`, case-insensitive on APFS). Convention: files matching that rule are **human-only** scratch and are never committed.
- `.gitignore` — ignores `.DS_Store`/`._*`/`.vscode`, `*.bak`, `logs/*` (except `.gitkeep`), the `_HUMAN-ONLY.md` human-only rule, and the `.workspace_mysql_secured` install marker. Plus three scoped blocks: the `configs/dot-obsidian/` per-vault runtime state (workspace.json, workspace-mobile.json, graph.json, file-recovery/, cache/, .trash/, *.bak); the sketchybar machine-local artifact (`configs/sketchybar/helpers/wifi_rssi`); and the largest block — the **`configs/dot-claude/` defensive allowlist** (ignore-all `configs/dot-claude/*`, re-allow the same safe config the submodule's own `.gitignore` does, then a credential denylist: `settings.local.json`, `**/credentials.json`, `**/.credentials.json`, `**/*.pem`/`*.key`/`*.p12`/`*.pfx`, `**/.env*`, `.claude.json*`). Inert while it's a submodule; only active if it ever stops being one. *(Per-config `.gitignore` files are not used; rules are consolidated here.)*
